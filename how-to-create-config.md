# 📖 Подробное руководство по созданию конфигурации

## 1. Что такое конфигурация в нашем приложении

Конфигурация — это **текстовый файл**, в котором описано, какие объекты (узлы) будут в проекте, какие у них параметры, и как они связаны друг с другом.
Файл делается в формате **JSON** (JavaScript Object Notation) — это простой способ записывать данные в виде:

* фигурных скобок `{ }` — описывают **объект** (набор свойств);
* квадратных скобок `[ ]` — описывают **массив** (список элементов);
* `ключ: значение` — одно свойство объекта.

Пример самого простого JSON:

```json
{
  "name": "Пример",
  "number": 123,
  "enabled": true
}
```

---

## 2. Как устроена конфигурация

Вся конфигурация — это **массив узлов**.
Каждый узел — отдельный объект со своими полями.
Записывается это так:

```json
[
  { ...описание первого узла... },
  { ...описание второго узла... }
]
```

---

## 3. Что такое "узел"

**Узел** — это один элемент вашей системы. Это может быть:

-   интерфейс (сетевой порт, модем);
-   протокол (Modbus, IEC 104 и т.д.);
-   группа параметров (папка);
-   переменная.

---

## 4. Минимальные требования к узлу

В каждом узле обязательно два поля:

| Поле   | Что значит                                                          | Пример                                    |
| ------ | ------------------------------------------------------------------- | ----------------------------------------- |
| `node` | **Уникальное имя** узла (просто уникальный идентификатор, не путь!) | `"eth0"`, `"device1"`                     |
| `type` | Тип узла (строго из списка)                                         | `"interface"`, `"protocol"`, `"variable"` |

Список допустимых типов:

```
interface, protocol, protocolSpecific,
interfaceSpecific, dataObject, variable, folder
```

Пример минимального узла:

```json
{
    "node": "eth0",
    "type": "interface"
}
```

---

## 5. Дополнительные поля узла

Кроме обязательных, можно указать:

| Поле              | Значение                                          | Пример                                       |
| ----------------- | ------------------------------------------------- | -------------------------------------------- |
| `label`           | Человекочитаемое имя                              | `"Ethernet 0"`                               |
| `icon`            | Иконка (состоит из `name` и `color`)              | `{ "name": "plug", "color": "green" }`       |
| `settings`        | Набор параметров (см. §7)                         | `{ ... }`                                    |
| `children`        | Дочерние узлы                                     | `[ { ... }, { ...} ]`                        |
| `bulkCreation`    | Массовое создание                                 | `{ "enabled": true, "presets": [4, 8, 16] }` |
| `validationRules` | Проверки на уровне узла                           | `[ { ... } ]`                                |
| `usedIn`          | Где используется: `"send"`, `"receive"`, `"both"` | `"both"`                                     |

---

## 6. Как описывать иконку

Иконка состоит из двух полей:

```json
"icon": {
  "name": "plug",
  "color": "green"
}
```

Возможные значения `name`:

```
folder, variable, plug, unplug, cable, fileDigit, fileStack, package, anchor, goose, lrEllipsis
```

Возможные значения `color`:

```
gray, red, orange, yellow, green, teal, blue, cyan, purple, pink, black
```

---

## 7. Параметры узла (`settings`)

**settings** — это набор настраиваемых параметров.
Каждый параметр имеет:

| Поле         | Значение                                              | Пример                                                   |
| ------------ | ----------------------------------------------------- | -------------------------------------------------------- |
| `type`       | Тип значения                                          | `"string"`, `"number"`, `"boolean"`, `"float"`, `"enum"` |
| `label`      | Название для пользователя                             | `"Скорость передачи"`                                    |
| `default`    | Значение по умолчанию                                 | `9600`                                                   |
| `enumValues` | Для перечислений (`enum`) — список возможных значений | `[{"value":"rtu","label":"RTU"}]`                        |
| `rules`      | Правила проверки (см. §9)                             | `[ { ... } ]`                                            |
| `showInTree` | Показывать ли параметр прямо в дереве узлов           | `true`                                                   |
| `visibleIf`  | Условие показа (см. §8)                               | `{ ... }`                                                |

Пример параметра:

```json
"speed": {
  "type": "number",
  "label": "Скорость передачи",
  "default": 9600,
  "rules": [
    {
      "validator": "range",
      "params": { "min": 1200, "max": 115200 },
      "message": "Допустимый диапазон: 1200-115200"
    }
  ],
  "showInTree": true
}
```

---

## 8. Логические условия `visibleIf` и `workIf`

Условия нужны, чтобы:

-   показывать параметр только при выполнении условий;
-   выполнять проверку только в определённых случаях.

**Важно:** переменные (что именно проверяем) всегда должны быть внутри **`find`**.

Пример: проверяем, что у родителя значение параметра `lengthOfASDU` равно `2`.

```json
"workIf": {
  "==": [
    {
      "find": [
        { "what": "lengthOfASDU", "where": "parent" }
      ]
    },
    2
  ]
}
```

Где:

-   `what` — имя параметра;
-   `where` — где искать: `"self"`, `"parent"`, `"siblings"`, `"ignorefolder"`.

---

## 9. Правила проверки (`rules`)

Проверки нужны, чтобы пользователь не вводил недопустимые значения.

Примеры:

1. **Обязательное поле**

```json
{ "validator": "required", "message": "Это поле обязательно" }
```

2. **Диапазон чисел**

```json
{
    "validator": "range",
    "params": { "min": 1, "max": 247 },
    "message": "Допустимые значения: 1-247"
}
```

3. **Регулярное выражение**

```json
{
    "validator": "regex",
    "params": { "regex": "^[A-Z0-9_]+$" },
    "message": "Только заглавные буквы, цифры и _"
}
```

4. **Уникальное значение**

```json
{
    "validator": "unique",
    "params": { "within": "siblings" },
    "message": "Значение должно быть уникальным среди соседних узлов"
}
```

---

## 10. Пример полного узла

```json
{
    "node": "modbus1",
    "type": "protocol",
    "label": "Modbus RTU",
    "icon": { "name": "package", "color": "blue" },
    "settings": {
        "address": {
            "type": "number",
            "label": "Адрес устройства",
            "default": 1,
            "rules": [
                { "validator": "required", "message": "Укажите адрес" },
                {
                    "validator": "range",
                    "params": { "min": 1, "max": 247 },
                    "message": "1-247",
                    "workIf": {
                        "==": [
                            {
                                "find": [{ "what": "enabled", "where": "self" }]
                            },
                            true
                        ]
                    }
                }
            ]
        }
    },
    "children": []
}
```

---

## 11. Как избежать ошибок

-   Всегда проверяйте, что у `node` уникальное имя в пределах всей конфигурации.
-   Пишем запятые **только** между элементами, но не в конце списка.
-   Следите за правильными кавычками — всегда `"двойные"`.
-   Для `enum` обязательно указывайте `enumValues`.
-   Условия (`visibleIf`, `workIf`) всегда пишем с `find`.
